# PointNet 3D Part Volume Prediction

A clean, modular PyTorch deep-learning pipeline that predicts the **volume of a 3D part** purely from its surface point cloud using a simplified **PointNet** architecture.

---

## Project Structure

```
part_volume/
│
├── data/                         ← Auto-generated by main.py
│   ├── pointclouds/              ← .txt files (2048 × 6: x y z nx ny nz)
│   └── labels.csv                ← filename, volume, shape_type
│
├── models/
│   ├── __init__.py
│   ├── pointnet.py               ← PointNet regressor architecture
│   └── best_model.pth            ← Saved best checkpoint (after training)
│
├── utils/
│   ├── __init__.py
│   ├── generate_data.py          ← Synthetic dataset generator
│   └── preprocessing.py          ← Dataset class + DataLoader factory
│
├── train.py                      ← Training pipeline
├── evaluate.py                   ← Evaluation + metrics
├── main.py                       ← Full pipeline entry point
├── requirements.txt
└── README.md
```

---

## Quick Start

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Run the full pipeline
```bash
python main.py
```

This will:
1. **Generate** 200 synthetic 3D shapes (boxes, spheres, cylinders)
2. **Train** the PointNet model for 75 epochs
3. **Evaluate** on the 20% test split (MSE, RMSE, MAE, R²)
4. **Inference demo** on a fresh shape never seen during training

---

## Model Architecture

```
Input: (B, 6, 2048)  — [x, y, z, nx, ny, nz]
         ↓
Shared MLP (Conv1d per-point):
    Conv1d(6  → 64)  + BN + ReLU
    Conv1d(64 → 128) + BN + ReLU
    Conv1d(128→ 256) + BN
         ↓
Global Max Pooling  →  (B, 256)
         ↓
Fully-Connected Head:
    Linear(256 → 128) + BN + ReLU + Dropout
    Linear(128 →  64) + BN + ReLU + Dropout
    Linear( 64 →   1) + Softplus    ← ensures positive volume
```

---

## Input Format

Each `.txt` file is a space-separated matrix of shape **(2048, 6)**:

| x | y | z | nx | ny | nz |
|---|---|---|----|----|----|
| float | float | float | float | float | float |

---

## Preprocessing

| Step | Operation |
|------|-----------|
| XYZ centering | Subtract centroid → zero-mean |
| XYZ scaling | Divide by max L2 norm → unit sphere |
| Normal enforcement | Re-normalise to unit vectors |
| Augmentation | Random point shuffle per epoch |
| Tensor format | Transpose to (6, 2048) for Conv1d |

---

## Training Details

| Hyperparameter | Value |
|----------------|-------|
| Loss | MSELoss |
| Optimiser | Adam |
| Learning Rate | 1e-3 |
| Weight Decay | 1e-5 |
| Gradient Clip | 1.0 |
| Scheduler | ReduceLROnPlateau (factor=0.5, patience=8) |
| Epochs | 75 |
| Batch Size | 16 |
| Train/Test Split | 80 / 20 |

---

## Evaluation Metrics

- **MSE** – Mean Squared Error
- **RMSE** – Root Mean Squared Error
- **MAE** – Mean Absolute Error
- **R²** – Coefficient of Determination

## Inference on a Custom Point Cloud

```python
import torch
import numpy as np
from models.pointnet import PointNetRegressor
from utils.preprocessing import normalize_pointcloud

# Load model
ckpt  = torch.load("models/best_model.pth", map_location="cpu")
model = PointNetRegressor()
model.load_state_dict(ckpt["model_state"])
model.eval()

# Load your point cloud  (2048, 6)
pc = np.loadtxt("path/to/your_part.txt", dtype=np.float32)

# Preprocess
pc = normalize_pointcloud(pc)
pc_tensor = torch.from_numpy(pc.T).unsqueeze(0)  # (1, 6, 2048)

# Predict
with torch.no_grad():
    pred_volume = model(pc_tensor).item()

print(f"Predicted Volume: {pred_volume:.6f}")
```
